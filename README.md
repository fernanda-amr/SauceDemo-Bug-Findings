# SauceDemo – Exploratory Bug Findings

This repository contains a manual QA case study focused on identifying functional and UI defects in the SauceDemo e-commerce application.  
The scope of this project included exploratory testing, negative scenarios, and end-to-end flow disruption analysis.

## Scope

### In-Scope:
- Exploratory functional testing
- Negative scenarios
- UI and behavioral observations
- Bug reporting and documentation
- Video evidence embedded in PDF reports

### Out-of-Scope:
- Automation testing
- Performance / Load testing
- API testing
- Requirements validation

---

## Test Approach

Manual exploratory and scenario-based testing focusing on:
- Error guessing
- Unexpected user actions
- Validation bypass attempts
- User flow interruptions

---

## Main Findings

Exploratory testing resulted in the discovery of multiple defects related to:
- Items cannot be added to cart
- Removal inconsistencies
- Checkout information not validated
- Blocked input fields (Last Name)
- Checkout flow bypass without required data
- Finalization flow not completing
